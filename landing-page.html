FAQ

Frequently asked questions

Can’t find your answer here? Ask on the WordPress.org forum →

How do I get an API key for AI messages?

It depends on which provider you want to use:

Google Gemini (free tier available): Visit Google AI Studio, sign in with your Google account, and click Create API key. Your key will start with AIza.

OpenAI (ChatGPT): Sign in at platform.openai.com and create a new secret key. Your key will start with sk-.

Anthropic (Claude): Create an account at console.anthropic.com and generate an API key. Your key will start with sk-ant-.

Once you have your key, go to Settings → Techanum Maintenance → API Settings, paste it in the API Key field, and save. The plugin auto-detects the provider based on the key prefix, so you can leave the AI Provider dropdown on Auto-detect.

Does it work with LM Studio, Ollama, or OpenRouter (local/custom LLMs)?

Yes. Select Custom (OpenAI-compatible) from the AI Provider dropdown, then enter:

• Custom Base URL — e.g. http://localhost:1234/v1 for LM Studio, http://localhost:11434/v1 for Ollama, or https://openrouter.ai/api/v1 for OpenRouter.

• Custom Model — the model identifier your endpoint expects, e.g. llama3, mistral, or openai/gpt-4o-mini.

The plugin appends /chat/completions automatically if the URL doesn’t already end with that path. For local models that don’t require authentication, you can enter any placeholder in the API Key field.

Can I use my Anthropic (Claude) API key?

Absolutely. Anthropic is a first-class supported provider. The plugin sends your prompt to the Anthropic Messages API (claude-3-haiku by default, with automatic fallback to claude-3-5-haiku if needed).

Either leave AI Provider on Auto-detect (the plugin recognises the sk-ant- prefix) or explicitly select Anthropic (Claude) from the dropdown. Paste your API key and save.

Will my site still work normally during maintenance mode?

Yes — only non-logged-in visitors (and logged-in users whose roles are not on the bypass list) will see the maintenance page. Administrators are always bypassed automatically.

Your WordPress admin dashboard, the REST API, and the WP-Cron scheduler all continue to function as normal behind the scenes. The maintenance page simply intercepts front-end requests and returns a proper 503 Service Unavailable response.

How do I schedule automatic maintenance windows?

Scheduled maintenance windows are a Pro feature. With Solo Pro or Business you can set a precise start date/time and end date/time — the plugin will enable the maintenance page automatically at the start time and turn it off again at the end time, with no manual intervention required.

The free version requires you to toggle maintenance mode on and off manually via the settings page.

What happens if the AI call fails?

The plugin always has a fallback. If no API key is configured, or if the API call fails for any reason (network timeout, rate limit, invalid key, etc.), visitors see the built-in fallback message: “We are performing scheduled maintenance. We will be back soon!”

You can also override this at any time by entering your own text in the Custom Maintenance Message field — your custom message always takes priority over the AI-generated one.

The failed call is not cached, so the next page load will retry the API automatically rather than serving a stale fallback for the full hour.

Will I lose my settings if I deactivate the plugin?

No. Deactivating the plugin only turns off maintenance mode — it does not delete any saved settings (logo, message, API key, role configuration). When you re-activate, everything is exactly as you left it.

If you want to remove all plugin data, you must delete the plugin entirely. A one-click “Delete all data on uninstall” option is planned for a future Pro release.

Can I define the API key in wp-config.php instead of the settings page?

Yes. As a secure alternative to storing the key in the database, define the following constant in your wp-config.php:

define( 'TECHANUM_ANTIGRAVITY_API_KEY', 'your-api-key-here' );

The settings-page value always takes priority; the constant is only used as a fallback when the settings field is empty. This is useful on staging/production environments where you manage secrets via server configuration rather than the database.

Quick Start Guide

Get your professional maintenance page live in under 5 minutes. No coding required.

  1. 1

    Install the plugin

    In your WordPress admin, go to Plugins → Add New Plugin and search for Techanum Maintenance. Click Install Now. Alternatively, download the .zip file from WordPress.org and upload it via Plugins → Add New → Upload Plugin.

  2. 2

    Activate the plugin

    Click Activate Plugin on the installation confirmation screen, or find Techanum Maintenance in your Plugins list and click Activate.

  3. 3

    Open the settings page

    Navigate to Settings → Techanum Maintenance in your WordPress admin menu.

  4. 4

    Enable maintenance mode

    Under the Maintenance Page section, tick the Enable Maintenance Page checkbox and click Save Settings. Your site is now in maintenance mode. Open it in a private/incognito browser window to preview what visitors will see.

  5. 5

    Customise your maintenance page

    Still on the settings page, you can:
    • Click Upload Logo to add your own branding.
    • Type a message in the Custom Maintenance Message textarea.
    • Check any roles under Excluded Roles that should still see the live site (e.g. Editor, Shop Manager).
    • Select roles under Admin Notices Management to suppress dashboard clutter for non-admin users.

  6. 6

    (Optional) Add an AI API key and generate a message

    Scroll to the API Settings section. Choose your provider from the AI Provider dropdown (or leave it on Auto-detect), paste your API key, and save. Then scroll back up to the Custom Maintenance Message field and click Generate with AI. The plugin will call your chosen AI, display the generated message in the textarea, and cache it for 1 hour going forward.

🔌 Using a local or custom AI endpoint?
Select Custom (OpenAI-compatible) from the AI Provider dropdown. Enter your base URL (e.g. http://localhost:1234/v1 for LM Studio, http://localhost:11434/v1 for Ollama, or https://openrouter.ai/api/v1 for OpenRouter) and the model name your endpoint expects (e.g. llama3, mistral, openai/gpt-4o-mini). The plugin appends /chat/completions automatically. This makes Techanum Maintenance compatible with virtually any OpenAI-compatible inference server, cloud or local.
